## 2. 产品目标与范围

### 2.1 产品目标

将当前依赖人工登录多台 Linux 服务器、启动程序、抓包、导出数据、执行 Coco 解析和统计脚本的测速流程，统一到一个 Web 平台中，实现：

- 测速资源集中管理和连通性检查；
- 测速方案、场景和运行记录的可复用管理；
- 通过 SSH 自动编排可自动化的远端步骤；
- 对机房接线等物理动作提供人工确认节点；
- 原始文件、解析结果、指标、结论和报告完整关联；
- 资源独占、任务排队、异常恢复和审计追踪。

### 2.2 首期业务

平台支持三类业务上下文：

- `fut_mm`：软核做市；
- `rem_two`：整合版二期；
- `rem_two_mm`：整合版二期做市。

三类业务共用平台框架，通过资源能力、场景类型和版本化配置表达差异。

### 2.3 首期范围

首期包含资源配置、权限管理、测速方案和场景、任务编排、人工确认、数据采集、Coco 解析、统计、结果复核、Web/Excel/PDF 报告和审计日志。

首期不固化具体交易所配置，不实现机房物理接线、网卡固化、设备断电重启等硬件动作，也不要求首期支持除测速以外的测试业务。

## 3. 用户、角色与安全

### 3.1 角色

| 角色 | 主要权限 |
| --- | --- |
| 管理员 | 用户、角色、业务、资源、系统配置、审计和全部任务管理 |
| 测试人员 | 创建和执行方案/场景、处理人工确认、查看和导出报告 |
| 访客 | 只读查看任务、指标和报告 |

账号由管理员创建，不开放公开注册。初始化账号为 `admin`，初始化密码为 `shengli123`；部署后应尽快修改该密码。管理员可重置其他用户密码，用户密码以哈希形式存储。

服务器密码、SSH 私钥、Token 等凭据使用应用密钥加密存储，页面列表和运行日志不得明文展示。认证采用 JWT（短期访问令牌 + 可撤销刷新令牌），所有 API 按角色和资源归属进行鉴权。

### 3.2 审计

记录登录、用户/角色变更、资源变更、任务状态迁移、人工确认、结论修改、产物删除和报告重新生成。审计日志至少包含操作者、时间、对象、动作、结果和请求来源。

### 3.3 完整日志系统

平台日志分为五类，统一使用 UTC 时间、服务名、环境、日志级别和请求追踪 ID（`trace_id`）：

| 日志类型 | 内容 | 主要用途 | 存储方式 |
| --- | --- | --- | --- |
| 应用日志 | API、Worker、Beat、数据库和队列异常 | 服务故障排查 | 结构化 JSON 文件，按天滚动并压缩 |
| 访问与安全日志 | 登录、登出、失败登录、鉴权失败、IP、User-Agent | 安全审计与攻击排查 | `log_records` + 不可变归档文件 |
| 测速运行日志 | 运行状态、步骤开始/结束、进度、重试、耗时和错误 | 查看任务执行过程 | `log_records`，关联 `run_id` 和 `step_id` |
| 远端命令日志 | SSH 命令、目标资源、退出码、stdout、stderr、开始/结束时间 | 重现和定位远端问题 | 数据库索引 + 任务产物文件 |
| 操作审计日志 | 用户、资源、方案、结论和报告的变更 | 责任追踪与合规 | `audit_logs`，禁止普通用户修改或删除 |

日志要求：

- 后端使用 Python `structlog` 输出结构化 JSON，禁止通过 `print` 输出业务日志；
- 每个 HTTP 请求、Celery 任务、WebSocket 会话和远端命令都生成或继承 `trace_id`；测速运行同时携带 `run_id`，步骤日志携带 `step_id`；
- 日志级别至少包括 `DEBUG`、`INFO`、`WARNING`、`ERROR`、`CRITICAL`，生产环境默认 `INFO`；
- 每条日志包含时间、级别、服务、模块、事件名、消息、追踪 ID、用户 ID（如有）、运行/步骤 ID（如有）和异常堆栈（如有）；
- SSH 密码、私钥、Token、JWT、Cookie 和命令中的敏感参数必须在写入前脱敏；命令日志记录模板和脱敏后的参数，不记录原始凭据；
- stdout/stderr 采用流式写入，支持大文件分片和 gzip 压缩；数据库只保存检索索引、摘要、大小、校验值和产物路径，避免日志膨胀业务表；
- Web 端按运行、步骤、级别、来源、时间和关键词筛选，支持实时跟随、暂停滚动、复制和下载；访客不能查看敏感命令参数；
- 访问日志和运行日志可由管理员检索，审计日志只允许追加，删除和导出必须再次写入审计日志；
- 默认应用/运行日志保留 90 天，审计日志保留 365 天，原始命令输出跟随任务产物策略保留；保留周期通过 `.env` 配置，清理前先归档并记录清理结果；
- 日志写入失败不得阻塞测速主流程；系统应降级到本地文件并产生告警，恢复后补传索引。

## 4. 资源管理

### 4.1 资源类型

- REM 柜台服务器；
- 模拟市场服务器；
- 发单工具服务器；
- 抓包节点；
- 解析分析节点（Coco）。

Coco 是测速链路中的数据解析与分析工具，读取 `pcapng`、订单数据、报价数据和账户交易编码等输入，生成解析 CSV 和测速分析结果。

### 4.2 统一资源字段

资源均支持名称、类型、所属业务、Linux 地址、SSH 端口、用户名、认证方式、远端路径、支持的市场/交易所能力、备注、启用状态、版本信息和健康状态。密码和私钥字段只允许安全写入，不回显原值。

资源管理功能包括列表、创建、查看、编辑、禁用、删除、SSH 连通性测试、健康检查和占用状态查看。删除有历史任务引用的资源时采用逻辑删除。

### 4.3 资源锁与排队

REM、模拟市场、发单、抓包和 Coco 节点在运行期间独占。提交任务时先校验资源可用性；冲突任务进入队列并展示预计等待状态。任务完成、失败、取消或超时后必须释放锁，服务重启时由锁租约和定时清理任务回收过期锁。

## 5. 方案、场景与运行

### 5.1 三级模型

- **测速方案**：业务上下文、默认资源组合和多个场景的可复用集合；
- **测速场景**：一个发单、撤单、报价或套利等具体动作集合；
- **测速运行**：某场景的一次实际执行，拥有独立步骤、日志、产物、指标和结论。

支持方案复制、场景复制、场景启停、配置版本和运行历史追溯。运行创建时保存方案/场景配置快照，后续修改不影响历史记录。

### 5.2 场景配置接口

场景提供通用、可扩展配置，不在 V2 固化具体测速参数：

- 资源选择和能力约束；
- 场景类型；
- 配置版本；
- 参数 JSON；
- 发单/撤单动作集合；
- 预期输入输出产物；
- 统计规则引用。

后续场景模板负责补充具体合约、端口、发单频率、绑核、配置文件和交易所参数。

## 6. 自动化测速工作流

### 6.1 状态机

```text
草稿
  → 资源排队
  → 环境预检
  → 等待人工接线确认
  → 抓包验证
  → 环境启动
  → 发单执行
  → 停止与数据收集
  → Coco 解析
  → 指标统计
  → 待人工复核
  → 完成
```

异常状态包括预检失败、执行失败、解析失败、用户取消和超时终止。每个状态迁移都记录操作者、时间、原因和关联日志。

### 6.2 标准流程

1. 选择业务、方案、场景和资源。
2. 对资源执行 SSH、目录、版本、进程、磁盘、负载等预检。
3. 锁定 REM、模拟市场、发单工具、抓包和 Coco 资源。
4. 提示测试人员完成机房接线，等待人工确认。
5. 启动抓包工具，执行试发单，检查四方向抓包文件是否生成。
6. 启动模拟市场和 REM 柜台。
7. 启动发单工具并执行场景动作。
8. 停止发单、REM、模拟市场和抓包程序，收集 `pcapng`、订单表、报价表和账户编码表。
9. 调用 Coco 解析程序，保存命令、日志、退出码和解析产物。
10. 执行统计分析，计算平均值、最大值、最小值、中位数、标准差、样本数和高频值占比等指标。
11. 根据基线或阈值生成规则预判。
12. 测试人员复核结果，填写最终结论、问题说明和备注。
13. 生成 Web 详情、Excel 明细和 PDF 报告，释放资源锁。

### 6.3 执行要求

- 非破坏性步骤支持有限次重试；
- 启动、停止、采集和解析必须记录远端命令、时间、退出码和日志；
- 取消任务时执行安全清理，避免遗留 REM、发单和抓包进程；
- 失败后可从可恢复步骤重试，不重复执行已成功且不可重复的步骤；
- 人工确认、状态变化和结论修改写入审计日志；
- 每个步骤实时写入结构化运行日志，远端 stdout/stderr 在命令结束前持续落盘并推送到前端；
- 日志记录异常时保留本地降级副本，并在任务详情中标记日志完整性状态；
- 任务超时由 Celery Beat 检查并触发清理与告警状态。

## 7. 结果、产物与报告

每次运行保存任务元数据、步骤时间线、结构化运行日志、远端命令和 stdout/stderr、原始抓包文件、数据库导出文件、Coco 原始解析结果、统计指标、规则预判、人工结论、Excel 明细和 PDF 报告。

报告支持 Web 在线查看、Excel 数据导出和 PDF 定稿下载。历史记录支持按业务、方案、场景、版本、时间、执行人和结论筛选。产物下载必须鉴权，删除和重新生成必须记录审计。

产物目录按 `业务/方案/场景/运行ID/步骤` 分层，数据库保存元数据和校验信息。原始文件与解析文件不可被普通用户直接覆盖。

## 11. 数据实体与 API

### 11.2 核心 API

- 登录、登出、当前用户；
- 用户、角色和权限管理；
- 资源 CRUD、连通性测试、健康检查；
- 测速方案和场景 CRUD、复制、启停；
- 创建、启动、暂停、恢复、取消测速运行；
- 查询运行状态、步骤日志、运行日志和实时事件；
- 按时间、级别、来源、关键词和 `trace_id` 查询应用/运行日志；
- 订阅运行日志 WebSocket、下载完整命令输出和导出审计日志；
- 查询指标、提交人工结论、生成和下载报告；
- 查询审计日志。

API 统一返回请求追踪 ID、错误码和用户可读消息；长任务只返回运行 ID，通过查询接口和 WebSocket 获取进度。

## 12. 部署与配置

首期按原生进程部署：Nginx、FastAPI、Celery Worker、Celery Beat、MySQL、Redis 均作为独立服务，由 systemd 托管。应用日志写入独立日志目录并由滚动策略压缩，运行命令输出写入任务产物目录，Redis Pub/Sub 负责实时广播，MySQL 保存可检索的日志索引和审计记录。数据库地址、Redis 地址、JWT、文件目录、日志级别、日志目录、保留周期和凭据加密密钥通过 `.env` 提供；仓库只提交 `.env.example`，不提交真实凭据。

## 13. 测试与验收

### 13.1 自动化测试

- 权限校验和越权访问；
- 状态机合法/非法迁移；
- 资源锁定、排队、释放和过期回收；
- 取消、超时和失败清理；
- SSH/SFTP 适配器异常处理；
- Coco 输入缺失、空结果和格式错误；
- 结构化日志字段、`trace_id`/`run_id`/`step_id` 关联和敏感信息脱敏；
- stdout/stderr 流式落盘、压缩、实时推送和日志写入降级；
- 日志查询、下载权限、审计日志不可变和保留周期清理；
- 统计指标计算；
- 报告生成和下载权限；
- WebSocket 断线重连与状态补偿。

### 13.2 集成与端到端测试

使用模拟 SSH 服务器、样例 `pcapng`、订单 CSV、报价 CSV 和账户编码表验证：

- 正常完成一条测速运行；
- 接线确认前暂停、确认后继续；
- 抓包为空时失败并保留诊断信息；
- Coco 报错后重试；
- 两个任务争用同一资源时按队列执行；
- 取消任务后远端进程被清理；
- 生成 Web、Excel、PDF 三种报告；
- 管理员、测试人员、访客的权限边界。

### 13.3 验收标准

- 测试人员无需手工登录远端服务器即可完成可自动化步骤；
- 每个步骤均可追踪状态、日志、耗时和错误原因；
- 能通过 `trace_id` 从 API 请求关联到 Celery 任务、运行步骤和远端命令；
- 运行期间可实时查看 stdout/stderr，任务结束后可下载完整日志；
- 日志中不出现密码、私钥、Token、JWT 或未脱敏的敏感命令参数；
- 应用日志、运行日志和审计日志具备不同的访问权限与保留周期；
- 同一资源不会被并发任务误用；
- 原始数据、解析数据、统计结果和最终报告可关联追溯；
- 失败任务不会遗留关键远端进程或资源锁；
- 三类业务共享平台框架并可通过配置表达差异；
- 新增交易所、工具或解析器无需重写核心工作流。
